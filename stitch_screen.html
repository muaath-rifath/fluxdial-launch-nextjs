Declarative Agent Engineering

Stop writing fragile dialogue trees. Define the agent's goal, provide the required JSON schema, and let the reasoning engine handle the conversational nuance. The output is always deterministic, structured data ready for your pipeline.

  • check_circle Sub-500ms voice-to-voice latency
  • check_circle Strict JSON schema adherence
  • check_circle Native Webhook & API integration
agent_config.yaml 
name: "Lead_Qualifier_v2"
engine: "flux-reasoning-fast"
voice_id: "en-US-professional-1"

system_prompt: >
  You are an SDR. Your goal is to qualify
  the lead based on BANT criteria.

required_schema:
  type: "object"
  properties:
    budget: { type: "number" }
    authority: { type: "boolean" }
    need: { type: "string" }
    timeline_months: { type: "integer" }
